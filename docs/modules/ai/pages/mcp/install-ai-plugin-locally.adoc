== Install Kobiton AI plugin locally
:navtitle: Install Kobiton AI plugin locally

Install Kobiton AI plugin locally to use it with a Standalone or Cloud with custom domain environment

== Before you start

[#_get_your_mcp_server_url]
== Get your MCP server URL

First, get the API base URL of your environment:

include::help-widget:partial$get-api-base-url.adoc[]

Add `/mcp` at the end of the retrieved API base URL.This will be the MCP server URL.See below for examples:

[cols="1,2,2",options="header"]
|===
|Environment |API base URL |MCP server URL

|Kobiton Cloud (default)
|`\https://api.kobiton.com`
|`\https://api.kobiton.com/mcp`

|Cloud with custom domain
|`\https://api-custom-domain.kobiton.com`
|`\https://api-custom-domain.kobiton.com/mcp`

|Standalone
|`\https://custom-domain.kobiton.local:3000`
|`\https://custom-domain.kobiton.local:3000/mcp`

|===

If you are using the default Kobiton Cloud URL (`\https://api.kobiton.com/mcp`), you can install the plugin from the online marketplace of your platform. Refer to guidance from the respective platform to install and authenticate.

* xref:ai:mcp/claude-code.adoc[]
* xref:ai:mcp/codex.adoc[]
* xref:ai:mcp/copilot.adoc[]
* xref:ai:mcp/gemini.adoc[]

If you are using a custom environment, continue with this guide.

== Download the Kobiton automate repository

Open a command-line tool on your machine:

- macOS: Terminal
- Windows: PowerShell or Command Prompt

If `git` is installed on your machine, clone the `automate` GitHub repository from Kobiton:

[source,bash]
----
git clone https://github.com/kobiton/automate.git && cd automate
----

If you do not have `git` installed, download the https://github.com/kobiton/automate/archive/refs/heads/main.zip[repository] as a `.zip` file, then extract it into the `automate-main` folder.

== Edit MCP configuration

Once you have obtained the <<_get_your_mcp_server_url,custom URL>>, edit the following file in the `automate` or `automate-main` folder:

[cols="1,2",options="header"]
|===
|Platform | MCP configuration file path

|Claude Code & Copilot
|`.mcp.json`

|Gemini
|`gemini-extension.json`

|Codex
|`.codex/.mcp.json`

|===

For all files above, replace the default value of `url` with the custom MCP server URL. Examples:

.Default
[source,json]
----
{
  "mcpServers": {
    "kobiton": {
      "type": "http",
      "url": "https://api.kobiton.com/mcp",
      "headers": {
        "X-AI-Tool-Name": "Claude"
      }
    }
  }
}
----

.Custom
[source,json]
----
{
  "mcpServers": {
    "kobiton": {
      "type": "http",
      "url": "https://api-custom-domain.kobiton.com/mcp",
      "headers": {
        "X-AI-Tool-Name": "Claude"
      }
    }
  }
}
----

== Install local plugin

NOTE: All the commands below need to be run in the Terminal shell, *not* the AI platform command-line interface (CLI).

[cols="1,2",options="header"]
|===
|Platform | Command to install

|Claude Code
|`claude plugin marketplace add /path/to/automate`

`claude plugin install automate@kobiton`

|Copilot
|`copilot plugin marketplace add /path/to/automate`

`copilot plugin install automate@kobiton`

|Gemini
|`gemini extensions install /path/to/automate`

|Codex
|`codex plugin marketplace add /path/to/automate`

`codex plugin add automate@kobiton`

|===
