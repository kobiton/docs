= Claude Code plugin reference
:navtitle: Claude Code plugin reference

The following reference lists available Claude Code plugins and their associated capabilities.

For setup and usage instructions, see xref:mcp/claude-code.adoc[Kobiton for Claude Code].

== dev-workflow
Full-lifecycle development pipeline for Kobiton: import tickets, plan multi-repo changes, implement, test, review, submit PRs, and monitor CI.

=== Commands
[cols="1,3", options="header"]
|===
| Command | Description
| run
| Run the full development pipeline end-to-end for a feature
| continue
| Auto-detect feature state and resume from the next pending step
| init
| Verify dev-workflow prerequisites (RepoSphere, gh, kubectl)
| import-issue
| Create a feature plan from a GitHub issue, JIRA ticket, or Confluence PRD
| proposal
| Create a multi-repo implementation plan
| proposal-wizard
| Interactive wizard for creating proposals step by step
| apply
| Implement change proposal tasks, tracking progress
| generate-tests
| Generate 4 test case sets for a feature (auto/manual, new/existing)
| select-tests
| Map code diffs to auto-test tags (3 execution modes)
| run-tests
| Trigger auto-test runs in a sandbox environment via GitHub Actions
| code-review
| Automated code review with confidence scoring via parallel agents
| pr-submit
| Open or refresh a single PR for one repo (idempotent)
| monitor-ci
| Watch one repo's PR CI; auto-fix retry loop on failure
| diagnose
| Diagnose test failures from test results, service logs, and k8s pods
| replan
| Revise implementation plan after failures or new information
| migrate
| Convert old .dev/ plan files to centralized features/ format
| list
| List all features (in-progress, completed)
| show
| Show detailed info about a specific feature
| status
| Show current dev-workflow state and health
|===

=== Agents

[cols="1,3", options="header"]
|===
| Agent | Description
| bug-detector
| Scan changes for logic errors, null handling, race conditions, and security vulnerabilities
| claude-md-auditor
| Audit changes for CLAUDE.md and project convention compliance
| code-architect
| Design multi-repo implementation blueprints using RepoSphere
| code-explorer
| Analyze codebase patterns, architecture, and dependencies
| code-simplifier
| Suggest clarity improvements without changing behavior
| comment-analyzer
| Analyze comments for accuracy, completeness, and maintainability
| history-analyzer
| Analyze git history for context on why code was written
| log-analyzer
| Read test and k8s pod logs to diagnose failures and identify root causes
| spec-validator
| Validate OpenSpec format and delta correctness
| test-analyzer
| Analyze test coverage quality and identify gaps
| test-case-generator
| Generate 4 test case sets per SDLC testing workflow (dev and TE modes)
|===
