= Search for a session
:navtitle: Search for a session

Learn how to search for a session so you can review general session information, view analytics, run validations, and more.

== Search for a session

In Kobiton, select *Sessions*.

image:session-explorer:session-search-context.png[width=1000,alt="A context of Session page in Kobiton portal"]

Use one or more xref:_search_methods[search methods] to search for your session.

[#_search_methods]
== Search methods

[#_run_a_saved_search]
=== Run a saved search

To run a previously-created xref:session-explorer:custom-search-queries.adoc[custom search query], select the *Saved Searches* dropdown and choose a search.

image:session-explorer:saved-search-context.png[width=1000,alt="Choose saved search"]

=== Select a date range

To display sessions within a specific date range, select *From* and *To*, then choose your dates.

image:session-explorer:date-range-context.png[width=1000, alt="Display sessions within the selected date range"]

[#_use_the_search_bar]
=== Use the search bar

To display sessions matching specific criteria, first select the search bar.

image:session-explorer:session-tags-context.png[width=500,alt="Sessions with Failed tag"]

You can manually enter specific xref:session-metadata.adoc[session metadata] or choose from the list of session tags:

==== Passed

Display sessions that passed their test criteria.

==== Complete

Display completed test sessions.

==== Running

Display sessions currently in progress.

==== Error

Display sessions that experienced errors during the session.

==== Failed
Display sessions that failed their test criteria.

==== Timeout

Display sessions that timed-out due to inactivity.

==== Terminated

Display sessions terminated mid-session.

==== Scriptless

Display sessions using Scriptless automation.

==== Manual

Display manual test sessions.

==== Appium

Display sessions using the Appium framework.

==== Espresso

Display sessions using the Espresso framework.

==== XCUITest

Display sessions using the XCUITest framework.

==== Mixed

Display sessions using both manual and automated testing during a session.
