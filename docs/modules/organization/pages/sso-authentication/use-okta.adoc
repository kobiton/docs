= Use Okta for SSO authentication
:navtitle: Use Okta

Use link:https://www.okta.com/[Okta] for Single Sign-On (SSO) authentication with SAML 2.0 so members of your organization can sign in to Kobiton with their Okta credentials.

== Create an IdP application

In Okta, select **Admin**.

image:organization:okta-admin-button.png[width=1000,alt="The Admin button in Okta home page"]

Okta may ask you to sign in again.

In the **Admin Console**, select **Applications**, then select **Applications** from the submenu.

image:organization:okta-applications.png[width=1000,alt="The Applications page with the Create App Integration option"]

Select **Create App Integration**.

In the **Create a new app integration** window, select **SAML 2.0**, then select **Next**.

image:organization:okta-create-app-sign-in-method.png[width=1000,alt="Choose SAML 2.0 in the select sign-in method step"]

In **General Settings**, enter an **App name**. Optionally, add a logo to customize your sign-in portal. Then select **Next**.

image:organization:general-settings.png[width=1000,alt="General Settings window"]

== Configure SAML settings and download the certificate

In the **Configure SAML Integration** window, use your xref:_get_kobiton_parameters[Kobiton SSO parameters] to configure Okta.

In **Single sign-on URL**, enter the Kobiton reply URL.

.Example
[source,text]
----
https://api.kobiton.com/v1/users/sso/saml/callback
----

For **Audience URI (SP Entity ID)**, enter the Kobiton entity ID.

.Example
[source,text]
----
https://api.kobiton.com/v1/users/sso/saml
----

For **Default RelayState**, enter the value from the Kobiton SSO basic configuration.

.Example
[source,json]
----
{"domainName": "Kobiton"}
----

image:organization:okta-fill-out-fields-context.png[width=1000,alt="Fields to fill in for Kobiton SSO"]

Optionally, select **Preview the SAML assertion** to review it before continuing.

Select **Next**.

On the feedback page, select an **App type**.

For internal apps, select **This is an internal app that we have created**.

Answer any optional feedback questions, then select **Finish**.

Okta opens the app configuration page.

[NOTE]
====
Profile and group attribute statements are in the *Attribute statements* section. To enter them, expand *Show legacy configuration*. These settings aren't required for the basic SAML setup flow unless your organization needs custom SAML attributes.
====

To download the certificate, go to the **SAML Signing Certificates** section. Under **Actions**, select **Download certificate**.

image:organization:download-okta-certificate.png[width=1000,alt="Select Download certificate for the Okta SAML signing certificate"]

== Get IdP parameters

Save the Okta IdP parameters you need to add to Kobiton.

On the Okta app configuration page, go to the **SAML Signing Certificates** section. Under **Actions**, select **View IdP metadata**.

From the IdP metadata, copy and save the following values:

* **Identity provider issuer**
* **Identity issuer URL**
* **Identity provider sign-in URL**
* **Identity provider sign-out URL**
* **Identity provider certificate**

Use the following metadata values. In the examples, angle-bracket text marks the variable parts of the URL; copy the full value from your metadata.

* For **Identity provider issuer**, enter Okta.
* For **Identity issuer URL**, use the `entityID` value from the `md:EntityDescriptor` element.

.Example
[source,xml]
----
entityID="http://www.okta.com/<your-okta-app-id>"
----

* For **Identity provider sign-in URL**, use the `Location` value from the `md:SingleSignOnService` element that ends in `/sso/saml`.

.Example
[source,xml]
----
Location="https://<your-okta-domain>.okta.com/app/<your-okta-app-instance>/<your-okta-app-id>/sso/saml"
----

* For **Identity provider sign-out URL**, use the sign-out URL Okta provides. If Okta doesn't provide a separate sign-out URL, use the same `/sso/saml` URL.
* For **Identity provider certificate**, use the `.pem` certificate file you downloaded from Okta.
